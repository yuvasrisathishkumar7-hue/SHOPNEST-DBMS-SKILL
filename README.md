# SHOPNEST – DBMS SKILL

## PROJECT TITLE
**ShopNest – E-Commerce Order Management Database System**

This repository contains the semester project for the **Database Management System (DBMS)** course. The project focuses on designing and implementing a centralized database system for an e-commerce platform. It includes requirement analysis, entity and relationship analysis, Entity Relationship (ER) Diagram design, and Relational Schema design.

---

# WEEK 1 – REQUIREMENT ANALYSIS

## OVERVIEW
This repository contains the Week 1 deliverables for the ShopNest – E-Commerce Order Management Database System. It includes the requirement analysis and documentation prepared before database design.

### FILES
- Requirement_Analysis_Report.pdf
- Business_Requirement_Document.pdf
- SRS_Document.pdf

### CORE ENTITIES
- Customer
- Category
- Product
- Supplier
- Order
- Order Details
- Payment
- Shipment
- Review

### OUTCOME
- Analyzed the business requirements of the proposed system.
- Identified the major business processes and stakeholders.
- Defined functional and non-functional requirements.
- Specified the project scope, assumptions, and constraints.
- Prepared the Software Requirement Specification (SRS).

---

# WEEK 2 – ENTITY AND RELATIONSHIP ANALYSIS

## OVERVIEW
This repository contains the Week 2 deliverables for the ShopNest project. The work focuses on identifying the entities, attributes, primary keys, foreign keys, database constraints, relationships, and cardinalities based on the Software Requirement Specification (SRS) prepared in Week 1.

### FILES
- Entity_Analysis_Report.pdf
- Entity_Relationship_Analysis.pdf

### OUTCOME
- Identified all system entities.
- Listed attributes for each entity.
- Defined Primary Keys (PK) and Foreign Keys (FK).
- Applied database constraints (NOT NULL, UNIQUE, DEFAULT, and CHECK).
- Analyzed entity relationships and relationship cardinalities.
- Established the conceptual foundation for ER Diagram design.

---

# WEEK 3 – ENTITY RELATIONSHIP (ER) DIAGRAM AND RELATIONAL SCHEMA DESIGN

## OVERVIEW
This repository contains the Week 3 deliverables for the ShopNest project. The work focuses on designing the Entity Relationship (ER) Diagram and converting the conceptual database model into a Relational Schema based on the Entity and Relationship Analysis completed in Week 2.

### FILES
- ER_Diagram.png
- Relational_Schema.pdf
- ER_Design_Report.pdf

### OUTCOME
- Designed the complete Entity Relationship (ER) Diagram.
- Represented all entities with their attributes.
- Identified Primary Keys (PK) and Foreign Keys (FK).
- Defined relationships between entities.
- Specified relationship cardinalities (One-to-One and One-to-Many).
- Converted the ER Diagram into a Relational Schema.
- Verified the mapping between entities and relational tables.
- Prepared the database design for SQL implementation.

---

# PROJECT SUMMARY

The ShopNest – E-Commerce Order Management Database System provides a centralized solution for managing customer information, products, suppliers, orders, payments, shipments, and customer reviews. The project follows a structured database design approach through requirement analysis, entity analysis, ER modeling, and relational schema design to ensure data consistency, integrity, and efficient business operations.
